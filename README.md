# network-security-lab-pfsense-wireshark
A cybersecurity lab demonstrating DoS attack detection and mitigation using pfSense firewall and Wireshark.
Title:
Network Security Lab Using pfSense Firewall and Wireshark

1. Project Overview
This project demonstrates how a pfSense firewall can detect and mitigate malicious network traffic in a controlled lab environment. A simulated Denial-of-Service (DoS) attack was generated from a Kali Linux machine targeting an Ubuntu server. Network traffic was analyzed using Wireshark, and pfSense firewall rules were configured to block the attack.

2. Lab Environment

System	Role and IP Addresses: Kali Linux	Attacker:	192.168.31.101, Ubuntu Server	Victim:	192.168.1.100, pfSense	Firewall,	Network Gateway and Wireshark	Packet Analyzer	Monitoring.

3. Attack Simulation
A DoS attack was simulated using hping3 from the Kali Linux machine.

Command:
hping3 -S --flood -p 80 192.168.1.100.

This command sends a high volume of TCP SYN packets to the target server.

4. Traffic Analysis Using Wireshark
Wireshark was used to capture and analyze network packets, revealing a sudden spike in traffic generated by the simulated attack.

5. pfSense Firewall Configuration
A firewall rule was configured in pfSense to block traffic from the attacker IP address targeting the victim server.

6. Attack Mitigation Results
After applying the firewall rule, pfSense successfully blocked the malicious traffic and prevented the attack from reaching the target system.
